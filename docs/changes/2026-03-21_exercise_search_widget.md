# 種目検索ウィジェット追加（ホーム画面）

## 概要

ホーム画面（カレンダー下）に「種目を探す」ウィジェットを追加する。
部位フィルタ・テキスト検索で種目を絞り込み、種目名をクリックすると既存の種目詳細ページ（`/exercises/:id`）へ遷移できる。

## 背景・目的

現在、種目詳細ページ（`/exercises/:id`）は存在するが、ホーム画面からの導線がなく一般ユーザーが種目の説明・補助筋情報を確認しにくい。
ホーム画面に検索ウィジェットを追加して、トレーニング中でも素早く種目情報にアクセスできるようにする。

### 動作仕様

| 項目 | 仕様 |
|------|------|
| 初期表示 | 全種目を一覧表示（`isActive=true` のみ） |
| 部位フィルタ | 「すべて」＋各部位ボタン（単一選択） |
| テキスト検索 | 種目名の前方一致（クライアントサイドフィルタ） |
| フィルタの組み合わせ | 部位フィルタ AND テキスト検索を同時適用 |
| 各種目の表示 | 種目名＋メイン部位名、クリックで `/exercises/:id` へ遷移 |
| 認証 | ログイン必須（ホーム画面自体が認証必須のため） |

### ExerciseSearchWidget コンポーネント仕様

**ファイル:** `frontend/src/components/exercise/ExerciseSearchWidget.tsx`

**状態管理（ローカル）:**
- `selectedBodyPartId: number | null` — 選択中の部位 ID（null = すべて）
- `searchText: string` — 検索テキスト

**使用する既存リソース:**
- `useExercises()` フック — 全種目取得（`frontend/src/hooks/useExercises.ts`）
- `useBodyParts()` フック — 部位一覧取得（`frontend/src/hooks/useBodyParts.ts`）

**フィルタロジック:**
1. `exercises` から `isActive=true` のものを抽出
2. `selectedBodyPartId` が指定されている場合、`exercise.bodyPart.id` が一致するものに絞る
3. `searchText` が入力されている場合、`exercise.name` が前方一致するものに絞る

**各種目行の表示:**
- `<Link to={`/exercises/${exercise.id}`}>` でラップ
- 表示内容: `種目名（メイン部位名）`

## 検証方法

1. バックエンド・フロントエンドを起動する
2. ログインしてホーム画面を開く
3. カレンダー下に「種目を探す」ウィジェットが表示されることを確認
4. 部位ボタン（例：「胸」）をクリックし、胸の種目のみに絞られることを確認
5. 検索ボックスに「スクワット」と入力し、スクワットのみ表示されることを確認
6. 部位フィルタと検索テキストを同時に指定し、AND条件で絞り込まれることを確認
7. 種目名をクリックして `/exercises/:id` の詳細ページに遷移することを確認
